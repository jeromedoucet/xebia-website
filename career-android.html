public class StringAdapter extends BaseAdapter {
 
       private final Context context;
 
       private final List<string> strings = new ArrayList<>();
 
       private StringAdapter(Context context, List<string> strings) {
 
           this.context = context;
 
           this.strings.addAll(strings);
 
       }
 
       @Override
 
       public int getCount() { return strings.size(); }
 
       @Override
 
       public String getItem(int position) { return strings.get(position); }
 
       @Override
 
       public long getItemId(int position) { return getItem(position).hashCode(); }
 
        
 
       @Override
 
       public TextView getView(int position, View convertView, ViewGroup viewGroup) {
 
           final TextView view = new TextView(context);
 
           view.setText(strings.get(position));
 
           return view;
 
       }
}

Chez nous, être développeur-se Android c'est :

- (Vraiment) Pratiquer l’agilité au quotidien (Sprint, Code Review, Pair Programming, etc.).
- Ne pas avoir peur de donner son avis et ce notamment, lors des réunions du pôle mobile.
- Accepter qu’on n’est jamais expert dans un domaine en continuant à faire de la veille de manière continue (Hands on, Tech Events, talks agiles et journée de partage de la connaissance mensuelle).
- Se frotter et se challenger sur d’autres sujets d’expertise que le sien (Android, Back-End, Front-End, Data, etc.).
- Rendre à la communauté ce qu’elle nous apporte en partageant son expérience à travers l’organisation de Tech-Events, la rédaction de billets de blog ou d’articles dans notre revue de presse hebdomadaire ou encore la participation à des conférences telles que DroidCon et Devoxx.
